# StudyFlow v4

시험 기간, 과목별 성적과 범위, 학원·생활 일정을 입력하면 학습 블록을 자동 생성하고 완료하지 못한 분량을 다시 배치하는 웹 플래너입니다.

이번 버전은 이전의 어두운 그라디언트·글래스 UI를 제거하고, 밝은 종이색 배경과 얇은 구분선을 사용하는 생산성 앱 형태로 전면 개편했습니다.

## 주요 기능

### 페이지형 입력 흐름

1. 시험 기간과 공부 가능 시간
2. 과목별 성적·목표·시험 범위·약점
3. 학원·운동·이동 등 반복 일정
4. 필요시간과 가용시간 비교 후 계획 생성

화면 주소는 해시 라우팅을 사용합니다.

```text
#/setup/exam
#/setup/subjects
#/setup/schedule
#/setup/review
#/dashboard/today
#/dashboard/week
#/dashboard/subjects
#/dashboard/tasks
```

### 계획 생성

- 시험 범위를 개념 이해, 문제 연습, 인출 연습, 누적 복습, 최종 점검으로 분할
- 이전 성적, 목표 점수, 자신감, 난이도, 범위량을 반영해 예상 학습시간 계산
- 시험 날짜와 취약도를 반영한 우선순위 계산
- 학원과 고정 일정 시간 제외
- 세션 사이 휴식시간 보장
- 희망 공부시간의 85%만 기본 계획으로 사용
- 기본 계획에 들어가지 않는 긴급 분량은 최대 공부시간 안에서 여유시간으로 배치
- 배치할 수 없는 분량은 정확한 시간으로 경고

### 실행과 재계획

- 오늘 가장 먼저 할 공부 강조
- 전체 화면 집중 타이머
- 완료, 일부 완료, 미루기
- 일부 완료 비율을 10% 단위로 선택
- 남은 분량만 새 일정으로 생성
- 지난 날짜의 미완료 일정은 다음 접속 시 오늘 이후로 자동 재배치
- 완료 처리와 재배치 작업 되돌리기
- 과목별 진행률과 공부법 추천
- 전체 일정 검색·상태·과목 필터
- Google Calendar에서 가져올 수 있는 ICS 파일 생성

### 저장과 설치

- 브라우저 `localStorage` 자동 저장
- 기존 `studyflow-ai-v1`, v2, v3 데이터 자동 변환
- 라이트·다크 테마
- PWA 매니페스트와 오프라인 캐시
- 모바일 하단 내비게이션

## 실행 방법

### 가장 간단한 방법

`index.html`을 브라우저로 열면 됩니다. 배포본은 `app.bundle.js`를 사용하므로 별도 설치 없이 실행할 수 있습니다.

### 로컬 서버 사용

```bash
python -m http.server 8000
```

브라우저에서 `http://localhost:8000`을 엽니다.

### GitHub Pages 배포

저장소에 폴더 안 파일을 모두 올린 뒤 다음과 같이 설정합니다.

1. GitHub 저장소의 **Settings**
2. **Pages**
3. **Deploy from a branch**
4. `main` 브랜치와 `/root` 선택

## 파일 구조

```text
adaptive-study-planner/
├── index.html             # 화면 구조와 템플릿
├── styles.css             # 전체 반응형 UI
├── app.bundle.js          # 브라우저 실행용 생성 파일
├── app.js                 # 화면·라우팅·상태 제어 소스
├── js/
│   ├── constants.js       # 기본값과 공통 상수
│   ├── utils.js           # 날짜·시간·문자열 유틸
│   ├── storage.js         # 저장과 구버전 데이터 변환
│   └── planner.js         # 학습량 계산과 일정 배치 엔진
├── tests/
│   └── planner.test.mjs   # 계획 엔진 단위 테스트
├── tools/
│   └── build.py           # 모듈을 app.bundle.js로 결합
├── manifest.json
├── icon.svg
├── sw.js
└── package.json
```

## 코드 수정 후 빌드

실제 로직은 `app.js`와 `js/` 폴더의 모듈을 수정합니다. 수정 후 브라우저용 번들을 다시 만듭니다.

```bash
npm run build
```

Python만 사용할 경우 다음 명령도 같습니다.

```bash
python tools/build.py
```

## 검사

```bash
npm run check
```

검사 항목:

- JavaScript 문법
- 시험 전 일정 배치
- 같은 날짜의 학습 블록 중복 여부
- 세션 사이 휴식시간
- 고정 일정 차감
- 반복 일정 충돌 감지
- 과목별 완료율 계산

## 현재 제한

- 사용자 계정과 기기 간 동기화는 아직 지원하지 않습니다.
- Google Calendar는 OAuth 직접 연동이 아니라 ICS 파일 내보내기 방식입니다.
- PDF·학습지 이미지 분석 기능은 아직 포함되지 않았습니다.
- 브라우저 저장 데이터를 삭제하면 입력 내용도 함께 삭제됩니다.
